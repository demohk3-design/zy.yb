# 期货 AI 研究助手

## 文件说明

- `angent.md`：AI 分析规则，核心是寻找“盘面资金确认 + 突发事件催化 + 明确止损边界”的做多机会。
- `sync_all.js`：自动同步公开数据，并生成多头候选评分。
- `market_data.json`：同步后的原始市场数据。
- `long_candidates.json`：盘面多头候选评分结果。
- `history/market_snapshots.json`：自动累计的历史快照，用于识别多日成交持仓信号。

## 使用方式

```bash
node sync_all.js
```

脚本会同时生成 `market_data.json` 和 `long_candidates.json`，并在终端显示全部品种的盘面评分、主要原因和主要风险。然后把结果交给 AI，按 `angent.md` 进行进一步分析。

## 当前自动数据覆盖

- 最新价、开高低收、昨结、均价、日内收盘位置；
- 成交量、持仓量；
- 多日持仓变化；
- 增仓抗跌、增仓上涨、减仓反弹、增仓破位等信号；
- 涨跌幅、日内位置；
- 部分现货价格和基差；
- 仓单最新值、1日变化、5日变化；
- 仓单折算手数；
- 仓单折手/持仓比例；
- 数据质量警告。

## 重要限制

免费公开数据无法稳定覆盖所有品种的真实突发事件和产业字段，例如：

- 社会库存；
- 厂库库存；
- 开工率；
- 产量；
- 成本线；
- 利润；
- 现货成交；
- 进口利润；
- 天气和政策事件的结构化数据。

因此，`sync_all.js` 只做第一层盘面筛选，最终判断仍需要 AI 联网核查最新突发事件和关键风控数据。

## 评分原则

脚本不会只因为涨了就推荐，也不会只因为基差大就推荐。它优先看盘面资金和仓单压力。

核心加分项：

- 收盘强于昨结；
- 收在日内高位；
- 收盘高于日内均价；
- 成交/持仓比活跃；
- 期货贴水且基差口径可靠；
- 仓单减少；
- 仓单压力较低。

核心扣分项：

- 仓单增加；
- 仓单折手占持仓比例偏高；
- 收在日内低位；
- 基差单位不可比；
- 缺少可比现货数据。
