# 期货研报极简分析规范 (Agent Guide)

此规范用于对任意期货品种进行大周期左侧与机构研报分析。
**【最高执行指令】**：在生成任何品种的分析前，Agent 必须强制使用搜索工具对底部“重点监控列表”中的**全部 11 家第一梯队期货公司**进行全景遍历检索与阅读。必须基于这 11 家机构的完整研报数据来综合提取成本、利润、支撑位与压力位，严禁仅查阅 3-4 家便凭空推演！

**【合约选择规则】**：若用户仅提供品种名称（如“螺纹”、“菜油”），Agent 必须首先通过检索确认当前市场成交量和持仓量最大的**主力合约代码**（如 `RB2610`，`OI2609`），并以此合约作为分析对象和文档命名依据。严禁随意指定非主力或非交易合约。

**【文档输出指令】**：每次生成的研报分析报告必须直接以 Markdown 格式写入并覆盖项目根目录下的 `<合约名称>.md` 文件（例如：`FG2609.md`）。

**【历史合约归档规则】**：已交割或失去主力流动性的历史合约 md 文件，应由 Agent 主动提示用户进行清理或归档，以保持工作区整洁。

所有输出必须严格按照以下模块进行极简排版，砍掉冗余文字。

## 重点监控与分析研报的期货公司列表

在撰写研报综合极简分析时，应优先检索和综合以下具有**独立深度调研与一手数据研究价值**的期货公司观点：

1.  **全品种核心主攻 (第一梯队 11家)：**
